## 二、采集性能优化

### 问题
原方案在一次采集中同时获取帧率、CPU、GPU、内存等全部指标，对设备造成额外性能压力，**无法反映真实帧率**。

### 解决方案
将单次采集改为**分阶段采集**（60秒）：

- **第1阶段（0-30s）**：仅采集帧率，不做其他测量，确保测到的是纯净的渲染性能
- **第2阶段（30-60s）**：采集 CPU/GPU/内存、DrawCall、设备信息等全部指标

前端性能分析页同步适配：帧率趋势图展示前30秒数据，资源占用图展示后30秒数据，一一对应。

## 三、前端体验优化

- 将独立的测试会话列表和性能分析入口**整合进项目管理详情页**，减少页面跳转，操作更直观
- **仪表盘**全面接入后端真实 API，新增强化趋势柱状图与状态分布饼图，无数据时友好提示
- **性能分析页**新增设备信息标签（CPU/GPU/内存/系统版本）
- 项目管理表格优化窄屏适配，操作栏固定不溢出
- 登录体验改善：刷新页面保持登录状态，Token 过期自动续期

---

## 四、自动化采集流程

将原来需要分别在 Unity 和前端手动操作的多步流程，整合为**一键自动化**：

- Unity 插件中点击「开始采集」→ 自动完成两阶段采集 → 自动上传至后端平台
- 支持设备令牌免密认证，无需在 Unity 端输入用户名密码
- 插件新增进度条和运行日志，全程状态可见

---

## 五、后端接口增强

- 新增**设备令牌认证接口**，Unity 插件用固定令牌换取 JWT
- 测试会话创建增加外键合法性校验，避免无效数据导致接口报错
- 分析报告接口扩展设备详细配置字段

## 六、数据流闭环

```
点击「开始采集」
  → 设备令牌认证 → 创建平台会话
  → 第1阶段 (0-30s)：纯净帧率
  → 第2阶段 (30-60s)：CPU/GPU/内存/设备信息
  → 自动上传后端
  → 前端仪表盘 / 性能分析页实时展示
```
