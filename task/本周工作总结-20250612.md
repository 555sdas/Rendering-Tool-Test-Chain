## 二、网页启动 Unity 本地测试

将上周的 Unity 插件手动采集方案，升级为 **前端一键启动 → 后端调度 → Unity 自动执行** 的全自动化流程：

- **前端**：项目详情页新增「Unity 本地测试」启动面板，支持选择 Unity 引擎版本、场景资源，配置采集间隔、帧率/指标采集时长，勾选渲染质量测试项（光照与阴影、材质与纹理、后处理、物理仿真）
- **后端**：新增 unity-runner 路由与 UnityRunnerService，负责读取资源配置、创建 TestTask/TestSession、生成 Unity 任务 JSON，并通过 `-executeMethod` 启动本地 Unity.exe
- **Unity 插件**：新增 XRBatchTestRunner 命令行入口，支持读取后端生成的任务 JSON，自动打开指定场景并配置 XRTestManager，采集完成后自动退出
- 测试会话按项目维度自动编号，测试结果与项目/会话绑定

---

## 三、多场景连续测试编排

实现了**同 Editor 内多个场景的连续自动测试**能力：

- **前端**：项目详情 Unity 测试区新增单场景/多场景 Tab 切换，提供多场景编排工作台（场景列表、Drawer 配置、批次监控、失败决策 Modal）；历史测试记录按 batch_id 折叠分组
- **后端**：新增 TestBatch / TestBatchItem / UnityProjectLease 模型与 unity_batches 路由（启动/查询/决策/停止），新增 UnityBatchService 批次编排服务，单场景启动接入工程租约防止并发冲突，批次看门狗监控与对账机制
- **Unity 插件**：新增 XROrchestrationRunner 实现多场景同 Editor 连续编排，支持域重载恢复、命令文件 retry/skip/abort
- **数据库**：新增 test_batches、test_batch_items、unity_project_leases 三张表及索引

## 四、前端功能增强

- 新增**系统设置页**，支持配置 Unity 可执行文件路径、项目目录、采集插件路径及默认测试场景
