## 2. 快速启动（Docker方式 - 推荐）

### 2.1 克隆项目并启动

```bash
# 1. 克隆项目到本地
git clone <repository-url> xr-test-platform
cd xr-test-platform

# 2. 复制环境变量模板
cp .env.example .env

# 3. 编辑 .env 文件（可选，使用默认值即可启动）
# 主要配置项：
# - POSTGRES_USER / POSTGRES_PASSWORD: 数据库账号密码
# - SECRET_KEY: JWT密钥，生产环境必须修改
# - NGINX_PORT: 外部访问端口，默认80

# 4. 一键启动所有服务
docker-compose up -d

# 5. 查看服务状态
docker-compose ps

# 6. 查看日志
docker-compose logs -f backend
```

### 2.2 首次启动后的初始化

```bash
# 进入后端容器执行数据库初始化
docker-compose exec backend python -c "from app.database import init_db; init_db()"

# 创建默认管理员账号（用户名: admin, 密码: admin123）
docker-compose exec backend python -c "
from app.database import SessionLocal
from app.models.user import User, UserRole, UserStatus
from app.core.security import get_password_hash
db = SessionLocal()
admin = User(
    username='admin',
    email='admin@xrtest.com',
    password_hash=get_password_hash('admin123'),
    full_name='系统管理员',
    role=UserRole.ADMIN,
    status=UserStatus.ACTIVE
)
db.add(admin)
db.commit()
print('管理员账号创建成功')
"
```

## 3. 本地开发环境搭建

### 3.1 安装Python依赖

```bash
cd backend

# 创建虚拟环境（推荐）
python -m venv venv

# Windows激活
venv\Scripts\activate

# Linux/Mac激活
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 3.2 配置本地数据库

```bash
# 使用Docker启动PostgreSQL和Redis（不启动后端和前端）
docker-compose up -d postgres redis

# 或者本地安装PostgreSQL后创建数据库
createdb xr_test_db

# 配置环境变量
export DATABASE_URL="postgresql://xruser:xrpass@localhost:5432/xr_test_db"
export REDIS_URL="redis://localhost:6379/0"
export SECRET_KEY="your-dev-secret-key"
export DEBUG="true"
```

### 3.3 数据库迁移

```bash
# 初始化Alembic（首次使用）
alembic init alembic

# 创建迁移脚本
alembic revision --autogenerate -m "init"

# 执行迁移
alembic upgrade head

# 回滚到上一个版本
alembic downgrade -1

# 查看当前版本
alembic current

# 查看历史版本
alembic history
```

### 3.4 启动开发服务器

```bash
# 方式1: 直接启动（热重载）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 方式2: 使用Python模块启动
python -m uvicorn app.main:app --reload

# 方式3: 生产模式（多进程）
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

## 6. 配置说明

## 8. 部署与运维

### 8.1 生产环境部署

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 重新构建镜像
docker-compose build

# 3. 执行数据库迁移
docker-compose run --rm backend alembic upgrade head

docker-compose up -d

curl http://localhost/api/v1/health
```

### 8.2 日志管理

```bash

docker-compose logs --tail=100 backend

# 查看特定时间段的日志
docker-compose logs --since="2024-01-01T00:00:00" backend
```

### 8.3 备份与恢复

```bash
# 备份数据库
docker-compose exec postgres pg_dump -U xruser xr_test_db > backup_$(date +%Y%m%d).sql

# 恢复数据库
docker-compose exec -T postgres psql -U xruser xr_test_db < backup_20240101.sql

# 备份上传文件
docker run --rm -v xr-test-platform_backend_uploads:/data -v $(pwd):/backup alpine tar czf /backup/uploads_$(date +%Y%m%d).tar.gz -C /data .

# 自动备份脚本（添加到crontab）
# 每天凌晨3点备份
0 3 * * * cd /path/to/xr-test-platform && ./scripts/backup.sh
```

## 9. 常见问题

### Q1: 启动时报数据库连接错误
```bash
# 检查PostgreSQL是否启动
docker-compose ps postgres

# 检查数据库是否存在
docker-compose exec postgres psql -U xruser -l

# 手动创建数据库
docker-compose exec postgres createdb -U xruser xr_test_db
```

### Q2: 前端无法访问API
```bash
# 检查CORS配置
cat .env | grep CORS

# 检查后端是否正常运行
curl http://localhost:8000/api/v1/health

# 检查Nginx配置
docker-compose exec nginx nginx -t
```

### Q3: 数据库迁移失败
```bash
# 查看当前版本
alembic current

# 手动标记版本
alembic stamp head

# 重置迁移（谨慎操作）
alembic downgrade base
alembic upgrade head
```

### Q4: 上传文件失败
```bash
# 检查上传目录权限
docker-compose exec backend ls -la /app/uploads

# 检查磁盘空间
docker-compose exec backend df -h

# 检查文件大小限制
cat .env | grep MAX_FILE_SIZE
```

## 10. 安全加固

### 10.1 生产环境必做

1. **修改默认密码**
   ```bash
   # 修改数据库密码
   # 修改 .env 中的 POSTGRES_PASSWORD
   
   # 修改JWT密钥
   # 修改 .env 中的 SECRET_KEY（至少32位随机字符串）
   
   # 修改管理员密码
   # 登录系统后在个人设置中修改
   ```

2. **启用HTTPS**
   ```bash
   # 准备SSL证书
   mkdir -p nginx/ssl
   cp your-cert.crt nginx/ssl/
   cp your-key.key nginx/ssl/
   
   # 修改nginx.conf启用HTTPS
   ```

3. **配置防火墙**
   ```bash
   # 仅开放必要端口
   sudo ufw allow 80/tcp
   sudo ufw allow 443/tcp
   sudo ufw deny 5432/tcp  # 不对外暴露数据库
   sudo ufw deny 6379/tcp  # 不对外暴露Redis
   ```

4. **定期更新依赖**
   ```bash
   # 检查安全更新
   pip-audit
   
   # 更新依赖
   pip install -r requirements.txt --upgrade
   ```
