# XR测试平台 - 项目总览与快速开始

## 1. 项目简介

XR测试工具链平台是一个面向XR（VR/AR/MR）应用的专业测试解决方案，支持：

- **性能测试**：帧率、帧时间、CPU/GPU/内存/显存监控
- **视觉检测**：清晰度、色彩、闪烁、抖动、拖影检测
- **图像对比**：SSIM、PSNR、Delta E差异分析
- **资源分析**：面数、Draw Call、材质、贴图内存统计
- **云AR测试**：端云协同、多人空间同步、交互响应
- **自动化测试**：脚本编排、批量执行、报告生成

## 3. 快速开始（5分钟启动）

### 3.1 环境准备

确保已安装：
- [Docker Desktop](https://www.docker.com/products/docker-desktop) (Windows/Mac)
- [Git](https://git-scm.com/)

### 3.2 一键启动

```bash
# 1. 克隆项目
git clone <repository-url> xr-test-platform
cd xr-test-platform

# 2. 复制环境配置
cp .env.example .env

# 3. 启动所有服务
docker-compose up -d

# 4. 等待服务启动（约30秒）
sleep 30

# 5. 初始化数据库
docker-compose exec backend python -c "from app.database import init_db; init_db()"

# 6. 创建管理员账号
docker-compose exec backend python scripts/create_admin.py

# 7. 访问系统
# Web界面: http://localhost
# API文档: http://localhost/api/v1/docs
# 默认账号: admin / admin123
```

### 3.3 验证安装

```bash
# 检查服务状态
docker-compose ps

# 测试API
curl http://localhost/api/v1/health

# 查看日志
docker-compose logs -f backend
```

## 5. 功能模块

### 5.1 用户权限管理

| 角色 | 权限 |

### 5.2 测试流程

```
1. 创建项目
   └── 设置项目名称、描述、目标平台

2. 配置测试
   └── 选择测试场景、设备、指标

3. 执行测试
   └── Unity插件采集数据 → 上传到后端

4. 分析数据
   └── 性能分析、视觉检测、图像对比

5. 生成报告
   └── 自动/半自动生成测试报告

6. 导出归档
   └── CSV/JSON/Excel/PDF导出
```

## 9. 常见问题

### Q: 如何修改默认端口？

编辑 `.env` 文件：
```bash
POSTGRES_PORT=5433      # 修改PostgreSQL端口
BACKEND_PORT=8001       # 修改后端端口
NGINX_PORT=8080         # 修改前端端口
```

### Q: 如何更新服务？

```bash
# 拉取最新代码
git pull origin main

# 重新构建并启动
docker-compose up -d --build

# 执行数据库迁移
docker-compose exec backend alembic upgrade head
```

### Q: 如何查看日志？

```bash
# 查看所有服务日志
docker-compose logs

# 查看特定服务
docker-compose logs -f backend

# 查看最近100行
docker-compose logs --tail=100 backend
```

### Q: 如何备份数据？

```bash
# 备份数据库
docker-compose exec postgres pg_dump -U xruser xr_test_db > backup.sql

# 备份上传文件
docker run --rm -v xr-test-platform_uploads:/data -v $(pwd):/backup alpine tar czf /backup/uploads.tar.gz -C /data .
```
